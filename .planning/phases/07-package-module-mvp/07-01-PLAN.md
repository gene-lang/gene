<objective>
Deliver the local-first package/module MVP: `package.gene` metadata parses into
one shared package model, `$pkg` exposes tested metadata, local/path `$dep`
dependencies produce deterministic lockfiles and diagnostics, package-aware
imports resolve through roots/source dirs/lockfiles, and public docs describe
the implemented boundary without claiming registry or full version solving.
</objective>

<threat_model>
Primary risk: package resolution becomes nondeterministic or docs imply a
registry-grade package manager before the local MVP is actually stable.

Mitigations:
- Reuse the existing `gene deps` lockfile and `src/gene/vm/module.nim` import
  machinery instead of creating a second resolver.
- Make `package.gene.lock` authoritative for dependency imports when present.
- Keep package metadata parsing shared between CLI/deps/runtime code.
- Add deterministic diagnostics for malformed manifests and dependency
  declarations.
- Document registry, remote index, complete semver solving, native trust, and
  signing as out of scope.
</threat_model>

<must_haves>
- `package.gene` parser supports `^name`, `^version`, `^source-dir`,
  `^main-module`, `^test-dir`, and `^dependencies` in flat pair and map forms.
- `$pkg` and `$app/.pkg` expose current package metadata in `run`, `eval`,
  `pipe`, `repl`, and imported modules where applicable.
- `gene deps install` writes `package.gene.lock` for local/path dependencies
  and `gene deps verify` validates the graph and hashes.
- Package-qualified imports honor package root, manifest source directory,
  local dependency overrides, direct import paths, and lockfile data.
- Focused package tests and `nimble testintegration` pass.
- Public docs state the MVP and future-only package features explicitly.
</must_haves>
