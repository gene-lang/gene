<objective>
Deliver the Phase 08 VM correctness harness: a compile-time-gated and runtime
opt-in checked VM mode, centralized instruction metadata, actionable invariant
diagnostics, clearer GIR compatibility failures, and parser/serdes/GIR stress
coverage for stable-core runtime behavior without changing optimized default
execution.
</objective>

<threat_model>
Primary risk: correctness checks accidentally become hot-path default behavior
or produce false confidence by claiming unsupported opcode/runtime edges are
fully checked.

Mitigations:
- Compile checked-mode helper bodies only under `-d:geneVmChecks`.
- Keep `VirtualMachine.checked_vm` false by default in `new_vm_ptr`.
- Make `--checked-vm` fail clearly when the binary was not built with
  `-d:geneVmChecks`.
- Centralize opcode metadata in `src/gene/types/instruction_metadata.nim`.
- Expose staged metadata gaps through `metadata_gap_kinds` instead of silently
  treating unknown dynamic behavior as covered.
- Keep refcount/lifetime validation to boundary sanity checks only.
- Add deliberate invalid-bytecode tests so diagnostics are proven against
  corrupted VM state, not just successful programs.
</threat_model>

<must_haves>
- `VirtualMachine` has a default-false checked-mode flag.
- Checked helpers are compiled only when `geneVmChecks` is defined and are
  no-ops otherwise.
- `run`, `eval`, and `pipe` support `--checked-vm` with a deterministic
  unsupported-build failure.
- Every `InstructionKind` returns metadata and staged gaps are visible.
- Checked failures include `VM invariant failed`, `pc=`, instruction kind, and
  a boundary label.
- GIR compatibility errors include expected/actual values and recompile/cache
  guidance.
- Parser, serdes, and GIR stress tests cover representative stable-core values
  and failure paths.
- Focused checks and `nimble testintegration` pass.
</must_haves>
